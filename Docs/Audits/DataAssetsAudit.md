# Gameplay DataAssets Audit

Generated by `GambitGameplayDataAuditCommandlet`. This audit is read-only: it loads real Unreal DataAssets through the Asset Registry, exports their authored values, and runs existing C++ validation helpers without saving or mutating assets.

## Command

From an Unreal command prompt or any shell where Unreal 5.7 is available:

```powershell
UnrealEditor-Cmd.exe "C:\Users\User\Documents\Unreal Projects\GrandpaGambit\GrandpaGambit.uproject" -run=GambitGameplayDataAudit -unattended -nop4 -nosplash -NullRHI
```

Useful parameters:

```text
ContentPath=/Game/GrandpaGambit/Data
JsonPath=Saved/Automation/GambitDataAssetsAudit.json
CsvPath=Saved/Automation/GambitDataAssetsAudit.csv
MatrixPath=Docs/ObjectCreation/ObjectMatrix.csv
-SkipMatrix
-FailOnValidationErrors
```

By default the commandlet returns success when export files are written, even if the audit contains validation errors. Use `-FailOnValidationErrors` for CI jobs that should fail on real DataAsset validation errors.

## Outputs

Default outputs:

```text
Saved/Automation/GambitDataAssetsAudit.json
Saved/Automation/GambitDataAssetsAudit.csv
```

The JSON contains:

- metadata: UTC timestamp, project name, project version if configured, project id, engine version, content path
- summary: total assets, counts by asset type, validation summary
- `assetsByType`: structured records grouped by asset type
- `matrixComparison`: conservative comparison against `Docs/ObjectCreation/ObjectMatrix.csv`

The CSV is flat and spreadsheet-friendly, with one row per exported asset plus optional matrix-only rows for matrix IDs that have no matching asset. Core columns include:

```text
AssetType, AssetPath, ObjectName, DisplayName, StableId, Rarity, Cost,
EffectHook, EffectType, TargetRuleId, NegativeCategories,
ValidationSeverity, ValidationMessages
```

## Covered Types

The commandlet scans these real DataAsset classes under `ContentPath`:

- `UGambitDiceDefinition`
- `UGambitDiceItemDefinition`
- `UGambitModuleDefinition`
- `UGambitConsumableDefinition`
- `UGambitItemEffectDefinition`
- `UGambitShopLootTable`
- `UGambitSharedPoolDefinition`

It reuses `GambitDataValidation` for dice, item, effect, shop loot table, and shared pool validation.

## Matrix Comparison

When `ObjectMatrix.csv` is available, comparison is done by stable ID:

- `Die` rows compare to `DiceId`
- `DiceItem`, `Module`, and `Consumable` rows compare to `ItemId`

The comparison reports:

- matrix rows expected by ID but not found as assets
- real assets whose stable ID is absent from the matrix
- type mismatches
- matrix rows marked `validated` while the real DataAsset has warnings or errors
- assets that exist while matrix progress is still `todo` or `blocked`

The comparison intentionally does not treat every design concept as a hard build failure. Rows with `todo` or `blocked` progress are reported as informational when missing.

### Interpreting Matrix Gaps

`MatrixAssetMissing` means a row exists in `ObjectMatrix.csv`, but no matching real asset was found by stable ID. For `todo` and `blocked` rows this is expected backlog/planning debt, not a validation failure. Do not change those rows to `validated` unless the audit exports a matching real DataAsset and the asset validation is clean.

`MissingFromMatrix` means a real comparable asset exists, but its stable ID is not present in `ObjectMatrix.csv`. Treat this as a triage prompt:

- add a row to `ObjectMatrix.csv` when the asset is production gameplay content that should be tracked as part of dice, shop, module, or consumable authoring
- document the exclusion when the asset is an example, validation fixture, debug/test content, legacy smoke asset, or technical support asset

The commandlet output is the source of truth for real DataAsset validation. `ObjectMatrix.csv` is the source of truth for production planning/tracking intent.

## Known Limits

- The audit does not parse `.uasset` files outside Unreal.
- The audit does not save packages or modify any DataAsset.
- ObjectMatrix comparison is ID/type based; it does not try to semantically compare every design note, parameter string, or planned mechanic.
- Effect assets are exported and validated, but `ObjectMatrix.csv` primarily tracks gameplay objects, so effect definitions are not treated as missing from the matrix unless matrix rows explicitly use `Type=Effect`.
- This pass does not address persistent per-player multi-target effect debt; it only exposes authored data and validation findings.
